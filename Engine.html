C++ game engine playground


A work in progress.
Started out as a minimalistic testing ground for performance with
lots of objects, and has since grown, in different directions.
It's really not something I want to display in terms of a finished
product, but maybe as a hint about interest, dedication, and/or
perseverance.


Environment: C++, SDL2, Vulkan, glfw
Link to this project on GitHub






January 2024
I decided to use OpenGL mainly because it's considered easier than Vulkan, at least to get started. And my thought was I'll just make a minimal OpenGL environment to get started and build from there. After getting it running I started to think about what I needed to do to integrate it into my engine, and realized that would actually be quite a bit more. So instead of putting too much time into learning OpenGl I decided to switch to Vulkan.
I have started to study small Vulkan implementations and, in my use case, it really doesn't seem like that big of a difference. (Famous last words..?) Really looking forward to integrating Vulkan!

January 2023
I had been wanting to drop SDL and work closer to the GPU, so I started working on an implementation with OpenGL, and the help of glfw.

April 2023
Added a physics system I created from different sources on the internet. I hadn't finished the integration when I started my internship, and chose to focus on Unity and C# from that point on, so this project was left in kind of a mess.

December 2022
Created an entity system with optional components for collision, sprites, etc.

November 2022
Did some testing with massive amounts of small moving objects on screen. Using SDL for the graphics.
